Step 2: Old Way vs. The Edge

Traditional apps live in a specific region (e.g., "us-east-1"). Cloudflare apps live everywhere.

🐢

The Origin Server

Users far away experience latency. If the server crashes, the site dies.

⚡

The Cloudflare Way

Code runs in 300+ cities. Users connect to the closest node. Millisecond latency.

Step 6: Workers (The Brain)

Database

Step 8: D1 (SQL at the Edge)

D1 is Cloudflare's serverless SQLite database. It stores structured metadata like course titles and IDs.

It's not a connection to a distant Postgres server. It's a database distributed near your Worker.
const result = await env.DB.prepare(
  'SELECT * FROM courses'
).all();

Caching

Step 12: Workers KV

We want to count views. D1 is great, but KV (Key-Value) is faster for simple counters.

KV is eventually consistent. It's perfect for things like user sessions, configuration, or view counts.
// Increment view count
let count = await env.VIEWS.get('aws-count');
await env.VIEWS.put('aws-count', ++count);
