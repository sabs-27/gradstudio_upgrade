Step 2: Centralized vs. Edge

Traditional databases live in one region (e.g., `us-east-1`). D1 replicates data closer to your users.

🐢

Legacy SQL

Single Region.

High latency for global users.

Connection pooling headaches.

⚡

Cloudflare D1

Global Read Replication.

SQLite based (file per DB).

HTTP-based API (no drivers needed).

Step 3: What is SQLite?

D1 is built on SQLite. Unlike Postgres or MySQL, SQLite is a database in a single file.

📁
➔
my_database.sqlite3

Why this matters: Since it's just a file, Cloudflare can easily copy (replicate) this file to hundreds of data centers around the world.

Step 10: Relational Data (JOINs)

D1 is SQL. We can perform JOINs between `simulations` and `tags` tables efficiently.

simulations
id: sim-001
⟷ JOIN ⟷
tags
tag: "compute"

SELECT s.title, t.tag_name
FROM simulations s
JOIN tags t ON s.id = t.simulation_id

Step 11: Hybrid Architecture

Not everything belongs in D1. High-velocity counters (like View Counts) might perform better in Cloudflare KV or Durable Objects, then sync to D1.

KV Store

Simple Key-Value.

Great for caching & simple counters.

Use for: View Counts (1250)

D1 SQL

Structured & Relational.

Great for complex queries.

Use for: Metadata & Search
