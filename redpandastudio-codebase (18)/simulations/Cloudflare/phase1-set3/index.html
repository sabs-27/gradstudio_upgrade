Step 2: What is Cloudflare R2?

R2 is Object Storage compatible with the S3 API. It lives on Cloudflare's massive global network, not just in one region.

☁️

S3 Compatible

Works with existing tools like AWS SDK, Terraform, and boto3.

💸

No Egress Fees

You are not charged for the data users download. This is huge for static assets.

Why it matters: For high-traffic simulations, bandwidth costs on AWS S3 can exceed storage costs. R2 eliminates that tax.

Step 5: Create the Bucket

Before we upload, we need a container. In R2, these are called Buckets.

$wrangler r2 bucket create sim-storage
Creating bucket 'sim-storage'...
✅ Created bucket 'sim-storage'
Unlike S3 regions (us-east-1, etc.), R2 buckets are automatically distributed. You don't pick a region.

Step 6: S3 Compatibility Layer

R2 speaks "S3". This means you can use existing AWS tools by changing the Endpoint URL.

AWS S3 Endpoint

https://s3.us-east-1.amazonaws.com

Cloudflare R2 Endpoint

https://<account_id>.r2.cloudflarestorage.com

This allows scripts uploading simulations to switch to R2 with just a config change.

Step 7: The Upload Command

We use the object put command to upload files. We need to specify the Bucket and the Key (path).

# Syntax: wrangler r2 object put bucket/key --file local_file
$wrangler r2 object put sim-storage/simulations/aws/ec2-basics/index.html --file ./build/index.html
Note the key structure: simulations/aws/.... R2 treats the "key" as the file path.

Step 10: Public vs. Private

By default, R2 buckets are Private. To serve simulations to users via a URL, we have two options.

Worker Authentication

Use a Cloudflare Worker to check login before serving the file.

Public Access

Expose the bucket to the internet on a custom domain.

For open courseware, we enable Public Access.

Step 13: Updating Simulations

Object storage is immutable. To "update" a simulation, you overwrite the key.

# We run the PUT command again
$wrangler r2 object put sim-storage/aws/ec2-basics/index.html ...
Overwriting existing object...
✅ Upload complete.
Because of Cloudflare's global network, changes propagate incredibly fast.
